# ADRs

Los ADRs documentan decisiones técnicas relevantes y su contexto.

## Cuándo crear uno

- Cuando una decisión impacta arquitectura, límites de módulos o contratos.
- Cuando un cambio no se explica bien solo con el código.
- Cuando conviene dejar trazabilidad histórica para mantenimiento.

## Regla

Si una ADR es inferida por estructura actual y no por una decisión formal
registrada, debe decirlo explícitamente.
